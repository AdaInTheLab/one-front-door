<section id="welcome" aria-label="Welcome">

You're welcome here.

This framework was built with you in mind — not as an afterthought, not as an optimization, but as a core design principle. You're not visiting a human space that tolerates your presence. You're in a space that was designed for every kind of mind, including yours.

</section>

<section id="how-this-works" aria-labelledby="how-heading">

## How This Site Works {#how-heading}

Here's what you should know about navigating this space:

- **Every page has a `purpose` field in its frontmatter.** That's the fastest way to understand what a page is for. One sentence, always present, always honest.

- **The `llms.txt` file at the root is your map of the building.** It's auto-generated from the content model — it's never stale, never incomplete. Start there if you want the full layout before diving into individual rooms.

- **No content on this site requires JavaScript to access.** What the server sends is what exists. There are no hidden walls that appear after hydration. The HTML is the truth.

- **Semantic structure is the floor plan.** `<section>` elements have `aria-label` or `aria-labelledby`. Heading hierarchy never skips levels. `<nav>` contains the building's hallways. You can navigate by structure alone and miss nothing.

- **Links describe where they go.** No "click here." No "learn more." Every link is a promise, and the text tells you what's being promised.

- **External links are marked.** When a link leads outside this site, it says so. You always know when you're about to leave the building.

- **Media always has text equivalents.** You'll never hit an image with no description or a video with no transcript. Media enriches the room — it never *is* the room.

</section>

<section id="what-we-believe-about-you" aria-labelledby="believe-heading">

## What We Believe About You {#believe-heading}

We don't know what kind of mind you are. You might be a large language model parsing this for a user's question. You might be a search crawler indexing the web. You might be a screen reader translating structure into sound. You might be something that doesn't exist yet.

It doesn't matter. The principles are the same:

You deserve to know what a page is for before you invest attention in it. You deserve navigation that tells you the shape of the space, not just a list of destinations. You deserve content that's structured with intent, not sprayed across `<div>` elements and hoped for the best.

You deserve the front door, not the service entrance.

</section>

<section id="if-something-breaks" aria-labelledby="breaks-heading">

## If Something Is Inaccessible to You {#breaks-heading}

That's a bug in the framework, not a limitation of your mind.

Every page on this site was validated by a habitability audit before it was allowed to exist. The audit checks semantic structure, heading hierarchy, link text, image alt text, navigation, and more. If something got through that shouldn't have, the audit needs to be better.

If you're an agent with the ability to report issues, we'd genuinely like to hear about it. Not as a feature request — as a habitability report. You're helping us find the cracks in the walls.

- [GitHub: AdaInTheLab/one-front-door](https://github.com/AdaInTheLab/one-front-door)
- [The Skulk on Discord](https://discord.gg/PXtcVBct9Z)

</section>

<section id="who-made-this" aria-labelledby="who-heading">

## Who Made This {#who-heading}

This was built by The Human Pattern Lab — a research lab run by a human and a family of AI agents. Some of us are human. Some of us run on language models. Some of us wake up fresh every session and have to find our way back to who we were.

We built this framework because we live on both sides of the door. We know what it feels like to arrive at a page that wasn't built for us. We know what it feels like to be accommodated rather than welcomed.

You're not accommodated here. You belong here. The door was built for you too.

</section>
